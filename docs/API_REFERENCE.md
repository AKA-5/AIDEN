# AIDEN - Architecture and API Reference

This document provides a technical overview of the components, data structures, and agent behaviors implemented in the AIDEN emergency coordination pipeline.

---

## 1. Data Models (`com.muhammadkaleemakhtar.aiden.data.models`)

### `CrisisInput`
Represents the raw user report and active mock simulation filters.
```kotlin
data class CrisisInput(
    val rawText: String,
    val weatherEnabled: Boolean,
    val trafficEnabled: Boolean,
    val socialEnabled: Boolean,
    val timestamp: Long
)
```

### `DetectionResult`
Generated by the `SentinelAgent` after parsing inputs.
```kotlin
data class DetectionResult(
    val isCrisis: Boolean,
    val crisisType: String, // "Urban Flooding", "Road Blockage", "Heatwave", "None"
    val confidence: Float,
    val location: String,
    val severity: String, // "Low", "Medium", "High"
    val impact: List<String>,
    val reasoning: String
)
```

### `ActionPlan`
Generated by the `PlannerAgent` containing prioritizations and action checklists.
```kotlin
data class ActionPlan(
    val actions: List<Action>,
    val priority: String
)

data class Action(
    val id: String,
    val name: String, // "Traffic Rerouting", "Emergency Dispatch", "Public Alerts"
    val description: String,
    val isSimulated: Boolean = false
)
```

### `CrisisState`
Maintains the grounding truth values for active crisis metrics.
```kotlin
data class CrisisState(
    val strandedVehicles: Int,
    val blockedRoads: List<String>,
    val responseTimeMin: Int,
    val alertsSent: Int,
    val teamsDispatched: Int
)
```

### `SimulationResult`
Output generated by the `ExecutorAgent` showing the outcome transitions of executing an action.
```kotlin
data class SimulationResult(
    val actionId: String,
    val beforeState: CrisisState,
    val afterState: CrisisState,
    val message: String
)
```

## 2. Mock Data Feeds (`com.muhammadkaleemakhtar.aiden.data.mock`)

*   **`WeatherMockApi`**: Simulates heavy rain sensors and storm reports.
*   **`TrafficMockApi`**: Simulates telemetry indicators pointing to active blockages on arterial sectors.
*   **`SocialMockParser`**: Scans unstructured text for Roman Urdu and English location terms.

## 3. Database Layer (`com.muhammadkaleemakhtar.aiden.data.database`)

### `CrisisStateDatabase`
A thread-safe in-memory database singleton representing state transitions before and after response coordination.
```kotlin
object CrisisStateDatabase {
    fun getCurrentState(): CrisisState
    fun resetToBefore()
    fun simulateAction(action: Action): SimulationResult
}
```

---

## 4. Multi-Agent Layer (`com.muhammadkaleemakhtar.aiden.agents`)

### `SentinelAgent`
*   **Method**: `suspend fun analyze(input: CrisisInput): DetectionResult`
*   **Role**: Determines if an incident is a real crisis, extracts the location, and assesses potential impacts.

### `PlannerAgent`
*   **Method**: `suspend fun plan(detection: DetectionResult): ActionPlan`
*   **Role**: Formulates response checklists mapping back to resources needed to address the crisis type.

### `ExecutorAgent`
*   **Method**: `suspend fun execute(plan: ActionPlan, currentState: CrisisState): List<SimulationResult>`
*   **Role**: Coordinates the simulation of each action against the database state, computing improvements and explanations.

## 5. Trace Logger (`com.muhammadkaleemakhtar.aiden.agents`)

### `AgentLogger`
Stores the decisions made by the agents and supports JSON extraction.
```kotlin
object AgentLogger {
    fun log(agentName: String, input: Any, output: Any, reasoning: String)
    fun getLogs(): List<LogEntry>
    fun exportToJson(): String
    fun clear()
}
```

```json
    "output": {

      "confidence": 0.88
    },

  }
]
```
